# Security Policy

## Reporting vulnerabilities

If you believe you have found a security issue, report it privately through GitHub Security Advisories or another private channel to the maintainer.

Please include:

- a clear description of the issue
- the affected component or workflow
- reproduction steps, if available
- the expected impact

## What to avoid

- Do not publish exploit details in a public issue.
- Do not share secrets, tokens, or private data in reports.

## Response goals

- confirm receipt promptly
- assess impact and scope
- coordinate remediation privately before public disclosure
