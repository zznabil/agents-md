# Security Policy

If you discover a sensitive issue in the repository contents, report it privately through GitHub Security Advisories or another private maintainer channel.

## Please include

- a short description of the issue
- the affected file or section
- reproduction steps, if any
- the impact you expect

## Please do not

- open a public issue for a sensitive report
- publish secrets, tokens, or private data
- disclose exploit details before coordination with the maintainer
